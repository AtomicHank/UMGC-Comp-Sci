A. Discussion of Approach

I approached the project in four passes:

1. Parser generalization
- Replaced the original +/- only subexpression parser with a dispatcher that handles all required expression forms inside parentheses: operand-only, unary postfix (~), binary (+ - * / % ^ < > &), ternary (?), and quaternary (#).
- Kept recursion through Operand::parse and SubExpression::parse so nested expressions still work naturally.

2. Evaluation model extension
- Added dedicated expression classes for each new arity:
  UnaryExpression
  BinaryExpression
  TernaryExpression
  QuaternaryExpression
- Implemented operator semantics exactly as specified:
  < = minimum, > = maximum, & = average
  ? and # conditional rules per assignment text.

3. Token/literal updates
- Updated variable scanning to accept underscores after the first character.
- Changed literal parsing from integer-only to floating-point (double) and used it in both expressions and assignments.

4. State and error correctness
- Reset symbol table before each statement evaluation to prevent value carryover.
- Added runtime exception handling for:
  Uninitialized variable access
  Duplicate variable initialization within the same statement

C. Lessons Learned and Potential Improvements

Lessons learned
- Designing the parser around expression arity (unary/binary/ternary/quaternary) made the grammar extension much easier than adding many one-off cases.
- Separating parse-time structure from runtime symbol checks improved clarity and made error handling straightforward.
- Moving to double everywhere early prevented mixed-type bugs later when adding floating literals.
- Explicit error types (EvaluationError) made runtime failures predictable and easier to test.

Potential improvements
- Add memory management with smart pointers (std::unique_ptr) to avoid manual new allocations.
- Add unit tests for parser and evaluator classes instead of only end-to-end file tests.
- Improve numeric handling for equality checks around zero in quaternary logic using tolerance (epsilon) if required.
- Add syntax validation and better diagnostics (line/column/operator context), even though the assignment says syntax checks are not required.
- Remove system("pause") and use standard console behavior for portability across platforms.
