                        RELATIONSHIP SUMMARY

IS-A RELATIONSHIPS (Inheritance - Generalization):
  - Circle IS-A TwoDimensionalShape IS-A Shape
  - Square IS-A TwoDimensionalShape IS-A Shape
  - Rectangle IS-A TwoDimensionalShape IS-A Shape
  - Triangle IS-A TwoDimensionalShape IS-A Shape
  - Sphere IS-A ThreeDimensionalShape IS-A Shape
  - Cube IS-A ThreeDimensionalShape IS-A Shape
  - Cone IS-A ThreeDimensionalShape IS-A Shape
  - Cylinder IS-A ThreeDimensionalShape IS-A Shape
  - Torus IS-A ThreeDimensionalShape IS-A Shape
  - ShapeGUI IS-A JFrame
  - ShapePanel IS-A JPanel

HAS-A RELATIONSHIPS (Composition/Aggregation):
  - Circle HAS-A radius (double)
  - Square HAS-A side (double)
  - Rectangle HAS-A width and height (double, double)
  - Triangle HAS-A base and height (double, double)
  - Sphere HAS-A radius (double)
  - Cube HAS-A side (double)
  - Cone HAS-A radius and height (double, double)
  - Cylinder HAS-A radius and height (double, double)
  - Torus HAS-A majorRadius and minorRadius (double, double)
  - ShapePanel HAS-A currentShape (Shape)
  - ShapeGUI HAS-A shapePanel (ShapePanel)
  - ShapeGUI HAS-A GUI components (JComboBox, JTextField, JButton, JLabel)

USES RELATIONSHIPS (Dependency/Association):
  - ShapeGUI USES ShapeFactory to create shapes
  - ShapePanel USES Shape subclasses for rendering
  - ShapeFactory USES Shape subclasses for instantiation

IMPLEMENTS RELATIONSHIPS (Interface Realization):
  - ShapeGUI IMPLEMENTS ActionListener
  - ShapeGUI IMPLEMENTS ItemListener

==================================================================================
                    DESIGN PATTERNS EMPLOYED
==================================================================================

1. FACTORY PATTERN
   - ShapeFactory.createShape() encapsulates object creation
   - Client code (ShapeGUI) doesn't need to know concrete classes
   - Makes adding new shapes easier (Open/Closed Principle)

2. TEMPLATE METHOD PATTERN
   - Abstract methods (getArea(), getVolume()) defined in base classes
   - Concrete implementations in subclasses
   - Common behavior in abstract classes, specific behavior in concrete classes

3. STRATEGY PATTERN (rendering)
   - Different rendering strategies for 2D vs 3D shapes
   - render2DShape() and render3DShape() switch based on shape type
   - Each 3D shape has specific rendering method (renderSphere, renderCube, etc.)

4. MODEL-VIEW-CONTROLLER (MVC)
   - Model: Shape classes (business logic and data)
   - View: ShapePanel (presentation)
   - Controller: ShapeGUI (user interaction and flow control)
