1. Using L'Hopital's rule and limits

Given:

f(n) = 6n^3 + 4n^2 + 2
g(n) = 5n^2 + 9

To test whether f(n) is in Omega(g), evaluate:

lim n->infinity f(n) / g(n)
= lim n->infinity (6n^3 + 4n^2 + 2) / (5n^2 + 9)

Apply L'Hopital's rule:

= lim n->infinity (18n^2 + 8n) / (10n)

Apply L'Hopital's rule again:

= lim n->infinity (36n + 8) / 10
= infinity

Because the limit goes to infinity, f(n) grows faster than g(n). Therefore:

f(n) is in Omega(g(n))

Now test whether g(n) is in Theta(f(n)):

lim n->infinity g(n) / f(n)
= lim n->infinity (5n^2 + 9) / (6n^3 + 4n^2 + 2)

Apply L'Hopital's rule:

= lim n->infinity 10n / (18n^2 + 8n)

Apply L'Hopital's rule again:

= lim n->infinity 10 / (36n + 8)
= 0

For two functions to be in the same Theta class, the limit of their ratio must be a positive finite constant. Here the limit is 0, not a positive constant. Therefore:

g(n) is not in Theta(f(n))

Conclusion:

- f(n) in Omega(g(n)) is true.
- g(n) in Theta(f(n)) is false.

- 3n log2 n

Reason:

- log4 n = (1/2) log2 n, so both are Theta(log n).
- sqrt(n) grows faster than logarithmic functions but slower than linear functions.
- 4n and 4n + 10 are both Theta(n).
- n log n grows faster than n but slower than n^2.
- n^2 + 6n and 2n^2 + 10n + 5 are both Theta(n^2).
- 2n^3 + n^2 + 6 is Theta(n^3).
- 2^n grows faster than every polynomial.


3. Recursion tree for sumSquares when n = 12

Method:

int sumSquares(int[] array, int first, int last) {
  if (first == last)
    return array[first] * array[first];
  int mid = (first + last) / 2;
  return sumSquares(array, first, mid) +
         sumSquares(array, mid + 1, last);
}

For n = 12, one recursion tree is:

sumSquares(0,11)
|- sumSquares(0,5)
|  |- sumSquares(0,2)
|  |  |- sumSquares(0,1)
|  |  |  |- sumSquares(0,0)
|  |  |  '- sumSquares(1,1)
|  |  '- sumSquares(2,2)
|  '- sumSquares(3,5)
|     |- sumSquares(3,4)
|     |  |- sumSquares(3,3)
|     |  '- sumSquares(4,4)
|     '- sumSquares(5,5)
'- sumSquares(6,11)
   |- sumSquares(6,8)
   |  |- sumSquares(6,7)
   |  |  |- sumSquares(6,6)
   |  |  '- sumSquares(7,7)
   |  '- sumSquares(8,8)
   '- sumSquares(9,11)
      |- sumSquares(9,10)
      |  |- sumSquares(9,9)
      |  '- sumSquares(10,10)
      '- sumSquares(11,11)

Number of nodes:

Each leaf corresponds to one array element, so there are n leaves. The recursion tree is a full binary tree, so:

total nodes = 2n - 1

For n = 12:

total nodes = 2(12) - 1 = 23

Execution time:

Every array element is processed once, and each internal node does only constant additional work. Therefore:

Theta(n)

Height of the tree:

Because the array is split about in half at each recursive step, the height in edges is:

ceil(log2 n)

If height is counted in levels instead, it is:

ceil(log2 n) + 1

For n = 12:

height in edges = ceil(log2 12) = 4

Memory:

Only one root-to-leaf path is active at a time, so the stack space is proportional to the tree height:

Theta(log n)

Iterative solution:

int sumSquaresIterative(int[] array) {
    int sum = 0;
    for (int i = 0; i < array.length; i++) {
        sum += array[i] * array[i];
    }
    return sum;
}

Comparison:

- Recursive version: Theta(n) time and Theta(log n) extra memory.
- Iterative version: Theta(n) time and Theta(1) extra memory.

The iterative solution is more space-efficient, while both versions have the same asymptotic running time.


4. Recurrence and Little Master Theorem

Modified recursion tree with work labels:

Each non-base activation does constant work c:

- test first == last
- compute mid
- add the two recursive results

Each base-case activation does constant work d:

- square one array element and return it

So the tree can be labeled as follows:

Level 0:               1 node   with work c      => row sum = c
Level 1:               2 nodes  with work c      => row sum = 2c
Level 2:               4 nodes  with work c      => row sum = 4c
...
Last internal level: about n/2 nodes with work c => row sum = (n/2)c
Leaf level:            n nodes  with work d      => row sum = nd

Initial condition:

T(1) = d

Recurrence equation:

For n > 1,

T(n) = 2T(n/2) + c

Critical exponent:

For T(n) = aT(n/b) + f(n):

- a = 2
- b = 2
- f(n) = Theta(1)

Therefore:

log_b a = log2 2 = 1

Apply the Little Master Theorem:

f(n) = Theta(1) = Theta(n^0)

Since 0 < 1, f(n) is polynomially smaller than n^(log_b a) = n.

Therefore:

T(n) = Theta(n)

Optimality:

Yes, the algorithm is asymptotically optimal. Any algorithm that computes the sum of squares of all n elements must examine every element at least once, so the problem has a lower bound of Omega(n). Since this algorithm runs in Theta(n), it matches the lower bound and is optimal.
