1. Recursion tree for recursive bubble sort on an array of length 4

Given:


void bubbleSort(int array[]) {
  sort(array, 0);
}

void sort(int[] array, int i) {
  if (i < array.length - 1) {
     bubble(array, i, array.length - 1);
     sort(array, i + 1);
  }
}

void bubble(int[] array, int i, int j) {
  if (j <= i)
    return;
  if (array[j] < array[j - 1]) {
    int temp = array[j];
    array[j] = array[j - 1];
    array[j - 1] = temp;
  }
  bubble(array, i, j - 1);
}


For an array of length 4 in the worst case, such as `[4, 3, 2, 1]`, the recursion tree is:

bubbleSort(array)
'- sort(array, 0)
   |- bubble(array, 0, 3)
   |  '- bubble(array, 0, 2)
   |     '- bubble(array, 0, 1)
   |        '- bubble(array, 0, 0)
   '- sort(array, 1)
      |- bubble(array, 1, 3)
      |  '- bubble(array, 1, 2)
      |     '- bubble(array, 1, 1)
      '- sort(array, 2)
         |- bubble(array, 2, 3)
         |  '- bubble(array, 2, 2)
         '- sort(array, 3)

Explanation:

- `bubbleSort` calls `sort(array, 0)`.
- Each `sort(array, i)` first performs one full bubbling pass from the end of the array down to `i`.
- Then it recursively calls `sort(array, i + 1)`.
- The `bubble` calls form a chain, not a branching tree.

Best-case difference:

The recursion tree shape is exactly the same in the best case. The only difference is that no swaps occur inside the `if (array[j] < array[j - 1])` statement. So the number of activations does not change, but the amount of swap work is smaller.

2. Node count, time, height, and memory

Number of nodes:

For an array of length `n`:

- There is 1 `bubbleSort` activation.
- There are `n` `sort` activations: `sort(0)` through `sort(n - 1)`.
- For each `i` from `0` to `n - 2`, `bubble(i, n - 1)` makes `n - i` activations, including the base case.

So the number of `bubble` activations is:

sum from i = 0 to n - 2 of (n - i)
= n + (n - 1) + ... + 2
= (n(n + 1) / 2) - 1

Total number of nodes:

1 + n + ((n(n + 1) / 2) - 1)
= n + n(n + 1) / 2
= (n^2 + 3n) / 2

So the total node count is:

`(n^2 + 3n) / 2`

Execution time:

The recursion performs the same amount of work as ordinary bubble sort. Therefore the execution time is:

`Theta(n^2)`

Height of the tree:

The longest root-to-leaf path is:

`bubbleSort -> sort(0) -> bubble(0, n - 1) -> bubble(0, n - 2) -> ... -> bubble(0, 0)`

That path contains:

- 1 `bubbleSort`
- 1 `sort`
- `n` `bubble` activations

So the height in nodes is:

`n + 2`

and the height in edges is:

`n + 1`

Thus the height is:

`Theta(n)`

Memory:

At one time, the deepest stack comes from one `sort` activation plus a chain of `bubble` activations. So memory usage is proportional to the height:

`Theta(n)`

3. Java class `SortedPriorityQueue`

public class SortedPriorityQueue {
    private final int[] queue;
    private int size;

    public SortedPriorityQueue(int capacity) {
        queue = new int[capacity];
        size = 0;
    }

    public void add(int value) {
        if (size == queue.length) {
            throw new RuntimeException("Priority queue is full");
        }

        int i = size - 1;
        while (i >= 0 && queue[i] > value) {
            queue[i + 1] = queue[i];
            i--;
        }
        queue[i + 1] = value;
        size++;
    }

    public int remove() {
        if (size == 0) {
            throw new RuntimeException("Priority queue is empty");
        }

        int result = queue[0];
        for (int i = 1; i < size; i++) {
            queue[i - 1] = queue[i];
        }
        size--;
        return result;
    }

    public boolean isEmpty() {
        return size == 0;
    }
}

Efficiency:

- `add`: worst case `Theta(n)` because elements may need to shift right.
- `remove`: worst case `Theta(n)` because elements shift left after removal.

This keeps the array sorted at all times and is straightforward with a plain `int[]`.


4. Efficiency of the sorting algorithm that uses `SortedPriorityQueue`

Algorithm:

void sort(int[] array)
{
  SortedPriorityQueue queue = new SortedPriorityQueue(100);
  for (int i = 0; i < array.length; i++)
    queue.add(array[i]);
  for (int i = 0; i < array.length; i++)
    array[i] = queue.remove();
}

Worst-case execution time:

Let `n` be the length of the input array.

Insertion phase:

- First add may cost `1`
- Second add may cost `2`
- ...
- nth add may cost `n`

Total insertion cost:

`1 + 2 + ... + n = n(n + 1) / 2 = Theta(n^2)`

Removal phase:

- First remove may shift about `n - 1` elements
- Second remove may shift about `n - 2`
- ...
- Last remove shifts `0`

Total removal cost:

`(n - 1) + (n - 2) + ... + 1 + 0 = n(n - 1) / 2 = Theta(n^2)`

Overall time:

`Theta(n^2) + Theta(n^2) = Theta(n^2)`

Comparison with Java's priority queue:

Java's `PriorityQueue` is heap-based.

- `add`: `Theta(log n)`
- `remove`: `Theta(log n)`

So sorting with Java's priority queue takes:

`n * Theta(log n) + n * Theta(log n) = Theta(n log n)`

Conclusion:

The array-based `SortedPriorityQueue` implementation is less efficient than the Java priority queue implementation.

- This implementation: `Theta(n^2)`
- Java priority queue version: `Theta(n log n)`
